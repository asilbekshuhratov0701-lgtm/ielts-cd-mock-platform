# 15 · Approved Extensions (E1–E14)

These 14 capabilities extend the foundation without replacing it. Each entry names the
schema/components it builds on and where it's specified.

## E1 · Exam Template System
Reusable exam blueprints (`ExamTemplate.kind`: Full Academic, Full GT, Listening/Reading/
Writing-Only, Mini, Custom) defining structure, timings, sections, defaults. Admin "Create
from template" instantiates an exam shell, then fills it from the Question Bank, AI import,
or by hand. → schema `ExamTemplate`; docs [03](03-database-erd.md), [11](11-admin-cms.md).

## E2 · Advanced Question Bank (content library)
Passages, audio, question groups and writing tasks become independently reusable via
`LibraryItem` + `isLibrary`/`category`/`difficulty`/`tagsJson`. Search, filter, categorize,
build exams from existing materials, and a **randomized-mock generator**. AI import deposits
extracted content into the bank. → docs [07](07-ai-import-pipeline.md), [11](11-admin-cms.md).

## E3 · Candidate Group Management
`CandidateGroup` (+ members): Morning/Evening/Weekend, IELTS 7+, VIP, custom. Assignments
target **individual / multiple / group / org** via `ExamAssignment.targetType/targetId`,
expanded to per-candidate rows. Group filtering + analytics. → API [04](04-api-spec.md).

## E4 · Universal Draft → Review → Publish workflow
One shared `ContentStatus` state machine + workflow service in `packages/core`, applied to
AI-imported exams, manual exams, writing evaluations, and future modules:
`Draft → Review → Preview → Approve → Publish → Assign → Archive`. **Hard gate:** nothing
reaches candidates without admin approval. → `packages/core/src/exam/state-machine.ts`.

## E5 · Exam Versioning
`Exam.rootId` / `parentVersionId` / `versionLabel`; duplicate, restore, archive, track
changes (`ExamVersionEvent` + audit). Published versions are **frozen** (edits fork a new
version). Candidate attempts stay linked to the exact version taken, so historical results
never shift. → docs [03](03-database-erd.md).

## E6 · Live Exam Center
Polling-based admin monitor driven by `SectionAttempt` + `SessionEvent`: active candidates,
current section, remaining time (from `deadlineAt`), connection status (heartbeat
staleness), recent autosaves, disconnects, auto-submissions, completed exams, warnings.
Realtime remains an optional future swap. → docs [11](11-admin-cms.md).

## E7 · Emergency Recovery module
Explicit `recovery` module (core + runner): deterministic resume after refresh / browser
crash / power loss / internet drop / restart / disconnect. Server is source of truth
(answers, `deadlineAt`, audio position derived from `startedAt`); `resumeToken` +
idempotent reconnect; client **IndexedDB** buffer re-syncs unsent answers. → docs
[13](13-cd-ux-spec.md), [14](14-backup-dr.md).

## E8 · IELTS CD Experience UX spec
Codifies official CD behaviours (play-once audio, split resizable Reading, navigator,
flagging, review screen, official timings, server-authoritative countdown, Writing word
counter, automatic transitions). → docs [13](13-cd-ux-spec.md).

## E9 · Media Library improvements
`MediaFolder` (nested) + `tagsJson` + version history (`parentMediaId`/`version`) + archive,
across audio/image/document management, with storage-usage reporting. → docs [11](11-admin-cms.md).

## E10 · Notification System
Typed in-app + email notifications (`Notification.channel`), generated by the worker on
events. Admin and candidate notification sets listed in [11](11-admin-cms.md).

## E12 · Backup & Disaster Recovery
Postgres PITR + daily snapshots + weekly backups; R2 versioning; documented restore
procedures, recovery testing, DR runbook. → docs [14](14-backup-dr.md).

## E13 · Writing Evaluation workflow
`WritingEvaluation.status`: `PENDING → ASSIGNED → IN_REVIEW → COMPLETED → PUBLISHED`.
Examiners work to **Completed** only; **admin approval required to Publish** (override
available). → docs [08](08-security-and-auth.md) (RBAC), [11](11-admin-cms.md).

## E14 · Administrative-automation principle
Cross-cutting rule: automate repetitive admin work, never remove admin authority. → docs
[00](00-overview-and-tech-stack.md) (Guiding principles).
